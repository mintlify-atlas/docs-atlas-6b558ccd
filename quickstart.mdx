## Setup Steps

<Steps>
  <Step title="Clone the Repository">
    First, clone the CaféIES repository to your local machine:

    ```bash
    git clone https://github.com/your-org/cafeies.git
    cd cafeies
    ```

    You should see the following project structure:
    ```
    CafeIES/
    ├── CafeIES.sln
    ├── CafeIES.API/          # Backend API
    ├── CafeIES.Admin/        # Blazor admin panel
    ├── CafeIES.MAUI/         # Mobile app
    └── CafeIES.Shared/       # Shared models
    ```
  </Step>

  <Step title="Configure the API">
    Navigate to `CafeIES.API/appsettings.json` and update the configuration:

    ```json CafeIES.API/appsettings.json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Server=.\\SQLEXPRESS;Database=CafeIES;Trusted_Connection=True;TrustServerCertificate=True;"
      },
      "Jwt": {
        "Key":      "CHANGE_THIS_KEY_TO_AT_LEAST_32_CHARACTERS!",
        "Issuer":   "CafeIES.API",
        "Audience": "CafeIES.App"
      },
      "Admin": {
        "Email":    "admin@cafeies.local",
        "Password": "Admin1234!",
        "Nombre":   "Administrador"
      }
    }
    ```

    <Warning>
      **Important Security Steps:**
      - Change the `Jwt.Key` to a random string of at least 32 characters
      - Update the `Admin.Password` to a strong password
      - For production, use environment variables instead of storing secrets in appsettings.json
    </Warning>

    **Configuration Explained:**
    - `DefaultConnection`: Your SQL Server connection string
    - `Jwt.Key`: Secret key for signing JWT tokens (keep this secure!)
    - `Admin`: Credentials for the initial admin account (created automatically on first run)
  </Step>

  <Step title="Create the Database">
    Open a terminal in the `CafeIES.API` directory and run Entity Framework migrations:

    ```bash
    cd CafeIES.API
    dotnet ef migrations add InitialCreate
    dotnet ef database update
    ```

    This will:
    - Create the `CafeIES` database
    - Set up all tables (Users, Products, Orders, etc.)
    - Seed initial data:
      - 5 default categories (Bocadillos, Ensaladas, Bebidas, Postres, Café)
      - 6 time slots for the three shifts (Morning, Afternoon, Evening)

    <Note>
      If migrations are already present, skip the `add` command and just run `dotnet ef database update`.
    </Note>
  </Step>

  <Step title="Start the API">
    Run the API server:

    ```bash
    dotnet run
    ```

    You should see output like:
    ```
    ✅ Admin creado: admin@cafeies.local
    ⚠️  Cambia la contraseña tras el primer login.
    Now listening on: https://localhost:7001
    ```

    The API will:
    - Apply any pending migrations automatically
    - Create the admin user if it doesn't exist
    - Start listening on `https://localhost:7001`

    **Test the API:**
    - Open `https://localhost:7001/swagger` in your browser
    - You should see the Swagger UI with all available endpoints
  </Step>

  <Step title="Configure API Base URL (MAUI)">
    Before running the mobile app, update the API base URL in `CafeIES.MAUI/MauiProgram.cs`:

    ```csharp CafeIES.MAUI/MauiProgram.cs
    // Update line 29-32 with your API port
    #if ANDROID
        var apiBase = "https://10.0.2.2:50658/";  // Change port to match your API
    #else
        var apiBase = "https://localhost:50658/";  // Change port to match your API
    #endif
    ```

    <Info>
      Android emulators use `10.0.2.2` to access the host machine's localhost. iOS simulators can use `localhost` directly.
    </Info>
  </Step>

  <Step title="Run the Mobile App">
    Open the solution in Visual Studio 2022:
    1. Set `CafeIES.MAUI` as the startup project
    2. Select your target platform (Android or iOS)
    3. Press F5 to build and run

    The app will:
    - Launch in your emulator/simulator
    - Show the login screen
    - Allow you to register as a student or scan an invitation code

    <Note>
      First build may take several minutes as it downloads dependencies and builds native assets.
    </Note>
  </Step>

  <Step title="Start the Admin Panel">
    Open a new terminal in the `CafeIES.Admin` directory:

    ```bash
    cd CafeIES.Admin
    dotnet run
    ```

    The admin panel will start on `https://localhost:7100`. Open it in your browser and:
    1. Login with the admin credentials from `appsettings.json`
    2. You'll see the dashboard with real-time order tracking

    <Warning>
      Change the default admin password immediately after first login!
    </Warning>
  </Step>
</Steps>

## Time Slots Configuration

By default, CaféIES seeds these time slots:

| Shift | Window 1 | Window 2 |
|-------|----------|----------|
| Mañana (Morning) | 07:30 - 08:00 | 11:00 - 11:30 |
| Tarde (Afternoon) | 13:45 - 14:00 | 17:00 - 17:30 |
| Noche (Evening) | 20:45 - 21:00 | 23:00 - 23:20 |

To modify these:
1. Go to admin panel → **Horarios**
2. Edit any time slot directly
3. Changes take effect immediately (no code changes needed!)

<Note>
  Teachers, staff, and admins are **not restricted** by time slots and can order anytime.
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="Database connection failed" icon="database">
    **Symptoms:** API fails to start with database connection error

    **Solutions:**
    - Verify SQL Server is running
    - Check the connection string in `appsettings.json`
    - For LocalDB, use: `Server=(localdb)\\mssqllocaldb;Database=CafeIES;Trusted_Connection=True;`
    - Test connection with SQL Server Management Studio
  </Accordion>

  <Accordion title="Mobile app can't connect to API" icon="mobile">
    **Symptoms:** Network errors in mobile app

    **Solutions:**
    - Android: Ensure you're using `10.0.2.2` instead of `localhost`
    - Verify the API port matches in `MauiProgram.cs`
    - Check that API is running and accessible at the configured URL
    - For Android, the development certificate is bypassed (line 39 in MauiProgram.cs:39)
  </Accordion>

  <Accordion title="JWT authentication fails" icon="key">
    **Symptoms:** 401 Unauthorized errors

    **Solutions:**
    - Ensure `Jwt.Key` in appsettings.json is at least 32 characters
    - Verify `Jwt.Issuer` and `Jwt.Audience` match in API and client
    - Check that the token hasn't expired (default 60 minutes)
    - Clear app data and login again
  </Accordion>

  <Accordion title="Admin user not created" icon="user-shield">
    **Symptoms:** Can't login to admin panel

    **Solutions:**
    - Check API logs for seeding errors
    - Verify `Admin` section in appsettings.json is correct
    - Manually check database: `SELECT * FROM Usuarios WHERE Rol = 99`
    - Delete the user and restart the API to recreate
  </Accordion>
</AccordionGroup>

## Running All Three Projects

For development, you can run all three projects simultaneously:

```bash
# Terminal 1 - API
cd CafeIES.API
dotnet run

# Terminal 2 - Admin Panel
cd CafeIES.Admin
dotnet run

# Terminal 3 - Mobile App (or use Visual Studio F5)
cd CafeIES.MAUI
dotnet build -t:Run -f net8.0-android
```

<Tip>
  Visual Studio 2022 can launch multiple projects at once. Right-click the solution → **Set Startup Projects** → **Multiple startup projects** → Set API and Admin to "Start".
</Tip>
