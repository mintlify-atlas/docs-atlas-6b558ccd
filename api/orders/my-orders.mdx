## Overview

Returns the most recent 20 orders for the authenticated user, ordered by creation date (newest first). Includes full order details with product information and line items.

## Authentication

<Warning>
  **Required:** Bearer token (JWT)
  
  **Roles:** All authenticated users (Alumno, Profesor, Personal, Admin)
</Warning>

## Query Parameters

This endpoint does not accept any query parameters. It automatically returns orders for the authenticated user based on the JWT token.

## Response

Returns an array of order objects, limited to the 20 most recent orders.

## Implementation Details

- **Query**: `SELECT TOP 20 * FROM Pedidos WHERE UsuarioId = @userId ORDER BY FechaCreacion DESC`
- **Eager Loading**: Includes related entities (User, LineaPedido, Producto) to minimize database queries
- **User Identification**: Extracts user ID from JWT token claim `ClaimTypes.NameIdentifier`
- **No Pagination**: Currently returns a fixed limit of 20 orders. Future versions may support pagination.

## Order Status Meanings

| Status | Value | Description |
|--------|-------|-------------|
| Pendiente | 0 | Order paid and submitted, waiting for cafeteria to acknowledge |
| EnPreparacion | 1 | Cafeteria is preparing the order |
| Listo | 2 | Order is ready for pickup at the counter |
| Entregado | 3 | Student has picked up the order |
| Cancelado | 4 | Order was cancelled (by user or admin) |

## Use Cases

- **Order History**: Display past orders in the mobile app
- **Reorder**: Allow users to quickly reorder items from previous orders
- **Receipt Access**: Users can view order details and totals
- **Support**: Customer service can reference order details
