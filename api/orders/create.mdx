## Overview

Creates a new order for the authenticated user. The endpoint validates:
- Schedule constraints (user's shift must be within an active time slot)
- Product availability and stock
- Calculates the total price based on current product prices
- Decrements product stock automatically
- Sends real-time notifications to cafeteria staff via SignalR

## Authentication

<Warning>
  **Required:** Bearer token (JWT)
  
  **Roles:** All authenticated users (Alumno, Profesor, Personal, Admin)
</Warning>

## Request Body

<ParamField body="Lineas" type="array" required>
  Array of order line items. At least one item is required.
  
  <ParamField body="ProductoId" type="integer" required>
    ID of the product to order
  </ParamField>
  
  <ParamField body="Cantidad" type="integer" required>
    Quantity to order. Must be between 1 and 20.
  </ParamField>
</ParamField>

<ParamField body="MetodoPago" type="enum" required>
  Payment method for the order.
  
  **Possible values:**
  - `0` - Tarjeta (Credit/Debit Card)
  - `1` - GooglePay
  - `2` - ApplePay
</ParamField>

<ParamField body="Notas" type="string" optional>
  Optional notes for the order (e.g., "sin lechuga"). Maximum 300 characters.
</ParamField>

## Response

<ResponseField name="Id" type="integer">
  Unique identifier for the order
</ResponseField>

<ResponseField name="NumeroPedido" type="integer">
  Sequential order number for the day (e.g., 42 displays as #042)
</ResponseField>

<ResponseField name="UsuarioNombre" type="string">
  Full name of the user who placed the order
</ResponseField>

<ResponseField name="UsuarioEmail" type="string">
  Email of the user who placed the order
</ResponseField>

<ResponseField name="FechaCreacion" type="datetime">
  UTC timestamp when the order was created
</ResponseField>

<ResponseField name="Estado" type="enum">
  Current order status:
  - `0` - Pendiente (Pending)
  - `1` - EnPreparacion (In Preparation)
  - `2` - Listo (Ready)
  - `3` - Entregado (Delivered)
  - `4` - Cancelado (Cancelled)
</ResponseField>

<ResponseField name="MetodoPago" type="enum">
  Payment method used (see request body for values)
</ResponseField>

<ResponseField name="Total" type="decimal">
  Total price of the order (calculated from current product prices)
</ResponseField>

<ResponseField name="Notas" type="string">
  Customer notes for the order
</ResponseField>

<ResponseField name="Lineas" type="array">
  Array of order line items
  
  <ResponseField name="ProductoId" type="integer">
    Product ID
  </ResponseField>
  
  <ResponseField name="ProductoNombre" type="string">
    Product name
  </ResponseField>
  
  <ResponseField name="Cantidad" type="integer">
    Quantity ordered
  </ResponseField>
  
  <ResponseField name="PrecioUnitario" type="decimal">
    Unit price at the time of purchase (frozen, won't change if product price changes)
  </ResponseField>
  
  <ResponseField name="Subtotal" type="decimal">
    Line subtotal (Cantidad × PrecioUnitario)
  </ResponseField>
</ResponseField>

## Example Request

  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "Lineas": [
      {

        "ProductoId": 7,

## Business Logic

1. **Schedule Validation**: Uses `HorarioService.PuedePedirAhoraAsync()` to check if the user's shift has an active ordering window
2. **Stock Management**: Products with `Stock = -1` have unlimited stock. Otherwise, stock is validated and decremented atomically
3. **Price Freezing**: Order line items capture the current product price at purchase time, so price changes don't affect existing orders
4. **Order Numbering**: Sequential numbering resets daily (e.g., #001, #002, etc.)
5. **Real-time Notifications**: Cafeteria staff connected to SignalR hub receive instant notification via `NuevoPedido` event
