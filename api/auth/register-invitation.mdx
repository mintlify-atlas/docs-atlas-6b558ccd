## Endpoint

<CodeGroup>
```bash Method and Path
POST /api/auth/registro/invitacion
```
</CodeGroup>

## Description

Registers a new teacher or staff member account using a valid invitation token. The account is immediately activated and the user is automatically logged in upon successful registration.

**Authentication Required:** No

## Request Body

<ParamField body="TokenInvitacion" type="string" required>
  The invitation token received from the administrator. This token determines the role assigned to the new user.
</ParamField>

<ParamField body="NombreCompleto" type="string" required>
  User's full name. Maximum length of 100 characters.
</ParamField>

<ParamField body="Email" type="string" required>
  User's email address. Must be a valid email format and unique in the system.
</ParamField>

<ParamField body="Password" type="string" required>
  User's password. Minimum length of 8 characters.
</ParamField>

## Response

<ResponseField name="AccessToken" type="string">
  JWT access token valid for a short period (typically 15 minutes). Used to authenticate API requests.
</ResponseField>

<ResponseField name="RefreshToken" type="string">
  Refresh token valid for 30 days. Used to obtain new access tokens without re-authentication.
</ResponseField>

<ResponseField name="Usuario" type="object">
  User information object containing the newly registered user's details.
  
  <ResponseField name="Id" type="integer">
    User's unique identifier.
  </ResponseField>
  
  <ResponseField name="NombreCompleto" type="string">
    User's full name.
  </ResponseField>
  
  <ResponseField name="Email" type="string">
    User's email address.
  </ResponseField>
  
  <ResponseField name="Rol" type="string">
    User's role in the system. Either `Profesor` (1) or `Personal` (2), depending on the invitation type.
  </ResponseField>
  
  <ResponseField name="Turno" type="null">
    Always `null` for teachers and staff (no shift restrictions).
  </ResponseField>
  
  <ResponseField name="Estado" type="string">
    Account status. Always `Activa` (1) for invitation-based registrations.
  </ResponseField>
</ResponseField>

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Registration successful, user automatically logged in |
| 400 | Invalid or expired invitation token, or validation errors |
| 409 | Email already exists in the system |

## Notes

- The invitation token determines whether the user is registered as a `Profesor` or `Personal`
- Unlike student registration, accounts created via invitation are immediately active (`Activa` status)
- The user is automatically logged in after successful registration
- Emails are automatically converted to lowercase before storage
- Passwords are securely hashed before being stored in the database
- Teachers and staff have no shift (`Turno`) restrictions
- Invitation tokens may have usage limits:
  - The `UsosActuales` counter is incremented with each successful registration
  - If the invitation reaches its `UsosMaximos` limit, it becomes inactive
  - Tokens can also expire based on their `FechaExpiracion`
- The invitation validation checks:
  - Token exists in the database
  - Token is active (`Activa` is true)
  - Token has not exceeded its usage limit
  - Token has not expired
